Encounter System

1. Although the collision presets are set to ignore everything except the player, an interface that only the player has is used to confirm the overlapping actor is the player.

The EncounterData property is a data asset that contains the enemy type, quantity and event trigger for each wave of the encounter.

2. Because designers were constantly iterating on the number of patrol routes and spawn locations, I used EQS instead of manually require designers to update arrays.

PopulatArraysInPatrolRouteManager calls RunEqsFindPatrolRoutes() which lives on another component. RunEqsFindActorSpawners lives on this component.

CanWaveSpawn and HasWaveSpawned are properties in the DataAsset not exposed to designers that are updated during gameplay. The for loop uses a reference of the wave data for each wave to reset the values to their default of false.

3. Designers assign gameplay tags 'wave.#' to the children of the ActorSpawner. Each element in the dataAsset's array has a 'wave' tag. The CurrentlyActiveWave is the tag of current wave.

The CurrentlyActiveEncounterTrigger is a box that encloses the entire encounter space. The EQS query finds all the actors of the object ype 'ActorSpawner' within the bounds of the encounter.

If the returned actors have a tag that matches the current wave, then they are added to the array of possible spawn locations.

4. This EQS takes in the EncounterTrigger as the context. The dimensions of the trigger component are used to set the bounds for a Box Multi Trace.

ObjectTypesToSearchFor is an instance editable array that is exposed to the EQS. In this example, the object type passed in is ActorSpawner.

Each ActorSpawner in the encounter is added to the AddGeneratedActors function, which is the array on actors the EQS returns.

5. For each encounter, could add up to ten waves to the array of a DA_Encounter. Each encounter trigger had one of these assets.

Quantity Spawned in Wave: total number of a given enemy type to spawn

Batch size: number of actors to spawn at a single spawn location (ActorSpawner class). If the Quanity=6 and the Batch-size = 2, then 3 batches, each with 2 enemies, would spawn.

If there were 3 ActorSpawners with a match wave.# gameplaytag, the 3 batches would all spawn at different locations. If there were fewer than 3, the array of possible spawn locations loops back to index 0.

Wave Trigger: designers could chose between Encounter-Start, Objective or Enemy-Population as s means to trigger a wave.

6. A unit test was used to automate the process of ensuring the correct data was entered.

When a new EncounterTrigger child was created, it couldn't be save until a DA_EncounterData was assigned to it's member variable.

When a new DA_EncounterData was created, it could be saved until the gameplay tags and values were properly set.
